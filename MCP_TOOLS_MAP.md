# MCP Tools Map

## Mnemostroma — 11 инструментов | 2026-04-10

Третья колонка оценивает соответствие философии памяти:

- 🧠 **Воспоминание** — возвращает сущность целиком с контекстом (флаги, атмосфера, история). Агент сам решает что важно.
- 🔍 **Навигация** — помогает найти что вспомнить, но не само воспоминание.
- ⚙️ **Сервисный** — управление системой, не извлечение памяти.

---

## Сессионный контекст

| Инструмент      | Логика                                                                                                | Соответствие памяти                                                                                                   |
| --------------- | ----------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------- |
| `ctx_semantic`  | Семантический поиск: embed(query) → MatrixSearch → rerank → top-N SessionBrief                        | 🔍 **Навигация** — возвращает briefs (сжатые), не полные сущности. Предшественник воспоминания, не само воспоминание. |
| `ctx_get`       | Прямой доступ по session_id из RAM, fallback в SQLite                                                 | 🔍 **Навигация** — точечный поиск по известному ID, не усилие вспоминания.                                            |
| `ctx_full`      | Полная запись сессии из SQLite: brief + why_log + content_full + теги + конфликт                      | 🧠 **Воспоминание** — возвращает сущность целиком как она была зафиксирована. Явный сознательный запрос.              |
| `ctx_anchors`   | Якоря субсознательного слоя: решения, факты, персоны, дедлайны — с key_facts, flags, decay_level. `type="deadline"` покрывает бывший ctx_urgent. | 🧠 **Воспоминание** — возвращает структурированную память с атмосферой (флаги, доступ, давность). |
| `ctx_precision` | Дословные артефакты из precision_log: ссылки, формулы, цитаты, данные                                 | 🧠 **Воспоминание** — прецизионное воспоминание, verbatim. Аналог «я точно помню формулу».                            |
| `ctx_search`    | Фильтрация RAM по пересечению тегов. Без ONNX, без SQLite                                             | 🔍 **Навигация** — требует знания точных тегов, не семантики. Ближе к каталогу чем к памяти.                          |
| `ctx_bridge`    | Структурированный пакет передачи контексту следующему агенту: intent + решения + конфликты + дедлайны | 🧠 **Воспоминание** — «что нужно помнить дальше». Передача рабочей памяти между агентами.                             |
| `ctx_recent`    | Сессии за последние N дней. by='created' или by='accessed'                                            | 🔍 **Навигация** — временной срез. Помогает найти недавнее.                                                           |

## Контентная ветка

| Инструмент        | Логика                                                                                                              | Соответствие памяти                                                                                                                          |
| ----------------- | ------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| `content_search`  | Семантический поиск по content_index (MatrixSearch + content_embedder). Возвращает метаданные блоков — без raw text | 🧠 **Воспоминание** — возвращает сущность (тип, теги, why_changed, версии). Агент осознанно запрашивает content_raw если нужен точный текст. |
| `content_get`     | Метаданные блока по content_id+version. RAM → SQLite fallback                                                       | 🧠 **Воспоминание** — полный контекст блока без тела. Агент видит «что это» прежде чем решить читать текст.                                  |
| `content_raw`     | Полный текст версии: lz4 decompress из RAM или SQLite                                                               | 🧠 **Воспоминание** — явный дорогой вызов. Аналог «вспомни дословно». Осознанное усилие.                                                     |
| `content_history` | Все версии блока включая rejected + rejected_reason. Только метаданные                                              | 🧠 **Воспоминание** — трассировка памяти: почему прошлые попытки были отклонены. Позволяет не повторять ошибок.                              |

## Демон-only (убраны из MCP)

Не агентские. Реализованы как функции демона / CLI-команды. См. `docs/DAEMON_TOOLS_PLAN.md`.

| Функция      | Было в MCP  | Как реализуется                                                     |
| ------------ | ----------- | ------------------------------------------------------------------- |
| `ctx_inject` | ✅ → ❌      | SDK-функция оркестратора, вызывается до запуска агента              |
| `ctx_status` | ✅ → ❌      | Демон пишет `~/.mnemostroma/status.json` каждые 30с                 |
| `ctx_growth` | ✅ → ❌      | CLI: `mnemostroma logs --growth` / cron-отчёт                       |
| `ctx_pulse`  | ✅ → ❌      | Демон пишет `~/.mnemostroma/pulse.json` каждые 5с                   |
| `ctx_sync`   | ✅ → ❌      | Демон: auto-flush в DBManager loop, + SIGUSR1                       |
| `ctx_dump`   | ✅ → ❌      | CLI: `mnemostroma dump` / SIGUSR2                                   |
| `ctx_evict`  | никогда     | Dissolver — подсознательный процесс демона                          |
| `ctx_load`   | ✅ → ❌      | Daemon IPC only. Перемещён 2026-04-10. Агент не меняет RAM-состояние |
| `ctx_active` | ✅ → ❌      | Удалён 2026-04-10. Покрывается `<memorycontext>` в system prompt   |
| `ctx_urgent` | ✅ → ❌ (→⟳) | Слит в `ctx_anchors(type="deadline")` 2026-04-10                   |

## Особый уровень: `marker()` — формирование памяти

`marker()` не является MCP-инструментом (не вызывается агентом напрямую), но архитектурно стоит **выше всех** инструментов в таблице.

```
Восприятие → [marker()] → Хранение → [ctx_full / ctx_anchors / …] → Воспоминание
```

| Компонент          | Роль                                                                                                      | Аналог в когнитивной психологии                            |
| ------------------ | --------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------- |
| `marker()`         | Классифицирует сырой текст → Entity / Emotion / Atmosphere / Discard. Присваивает важность, время, флаги. | **Кодирование** (encoding) — сырой опыт становится памятью |
| MCP 🧠-инструменты | Извлекают закодированные объекты целиком                                                                  | **Воспоминание** (retrieval) — усилие → полная сущность    |
| MCP 🔍-инструменты | Помогают найти нужный объект                                                                              | **Поиск в индексе** (memory search)                        |

Без маркера нет памяти — есть только поток текста. Маркер — это граница между «было сказано» и «запомнено».
