# Mnemostroma

### Слой памяти для AI-агентов

> *μνήμη (mnḗmē, память) + στρῶμα (strôma, слой) — субстрат, на котором всё держится.*

---

Открываешь новый чат. Объясняешь всё заново.
Модель понятия не имеет, что ты решил на прошлой неделе.
Что заблокировано. Что отменено. Что важно.

Ты общаешься не с агентом. Ты общаешься с золотой рыбкой, у которой PhD.

**Mnemostroma это исправляет.**

Сидит между тобой и твоим AI — тихо, незаметно, всегда включена.
Ты работаешь как обычно. Mnemostroma наблюдает, учится, запоминает.

Следующая сессия? Агент уже знает контекст.
Без промпт-хаков. Без копипасты логов. Без «как я уже говорил».

---

## Что делает

При каждой работе с AI-агентом Mnemostroma:

- **Ловит важное** — решения, ограничения, ключевые факты — автоматически
- **Сжимает умно** — не транскрипт, а дистиллированная память
- **Подсовывает когда нужно** — без твоих вопросов
- **Забывает плавно** — старое тускнеет, критичное живёт вечно
- **Работает офлайн** — твоя память, твоя машина, без облака

---

## Архитектура в одном предложении

Двухпоточный async pipeline (Observer + Content) поверх трёх слоёв памяти,
HNSWlib ANN-поиск, ONNX INT8 инференс, SQLite WAL холодное хранилище —
~600MB RAM, ~20ms retrieval, полностью офлайн.

---

## Как работает

```
Твой агент
    │
    ├── OBSERVER (async sidecar — пишет)
    │     Слушает весь I/O, извлекает сущности, векторизует, оценивает
    │     Агент сам никогда не пишет — Observer делает всё незаметно
    │
    ├── ИНСТРУМЕНТЫ АГЕНТА (только чтение)
    │     ctx.active()    → текущий контекст      <0.01ms
    │     ctx.semantic()  → поиск по смыслу         ~20ms
    │     ctx.search()    → поиск по тегам         <0.1ms
    │     ctx.bridge()    → пакет передачи сессии  <0.01ms
    │
    └── КОНТЕНТНАЯ ВЕТКА (версионированные артефакты)
          Код, главы, конфиги — с diff'ами и why_changed
```

**Агент никогда не пишет память.** Он только читает и действует.

---

## Модель памяти

Mnemostroma не архивирует — она **растворяет**.

```
День 1:    Полная детализация — brief, якоря, прецизионные данные, вектор
Неделя:    Детали тускнеют — precision уходит в SQLite
Месяц:     Brief + теги + якоря
Год:       Brief + вектор
Десятилетие: Только вектор — форма памяти без содержания
```

Что используешь — остаётся ярким. Что нет — постепенно растворяется.
Принципы не растворяются никогда. Решения живут долго. Телефоны истекают.

Это не база данных с TTL. Это так работает человеческая память.

## Компоненты

| Компонент          | Роль                                           | Аналогия                              |
| ------------------ | ---------------------------------------------- | ------------------------------------- |
| **Observer**       | Строит смысловой контекст из I/O агента        | Слуховая система                      |
| **Dissolver**      | Постепенно снижает разрешение памяти           | Кривая забывания                      |
| **Tuner**          | Детектирует конфликты, дрейф, устаревшие якоря | Настройщик рояля — слушает диссонанс  |
| **Conductor**      | Bootstrap, event loop, health, RAM-бюджет      | Дирижёр                               |
| **Session Bridge** | Передача контекста между сессиями              | Проснуться и вспомнить вчерашний день |
