## Бывшие 🔴 КРИТИЧЕСКИЕ → пересмотрены

### WP-01 — Feedback Loop
**v2.0:** 🔴 → ⚠️ Частично
**v3.0:** ✅ **ЗАКРЫТ**

**Пересмотр:** Implicit feedback v1.5 (USE/DEEP_USE/IGNORE/REVISIT) — это биологически корректная модель реконсолидации. Человеческая память тоже не имеет explicit feedback API. Мозг наблюдает за последствиями действий и корректирует — именно это делает v1.5.

| Человеческий механизм | Аналог в Mnemostroma |
|----------------------|----------------------|
| Часто вспоминаю → укрепляется | USE/REVISIT → implicit_score растёт |
| Не вспоминаю → тускнеет | Нет сигналов → Dissolver растворяет |
| Столкнулся с противоречием | Conflict Detector → conflict_flag |
| Контекст не помог → переспрашиваю | IGNORE сигнал (<5s re-query) |

Explicit feedback (v2.0) = convenience API для разработчиков, не архитектурная необходимость.

### WP-02 — Потеря данных при краше
**v2.0:** 🔴 → ⚠️ Частично
**v3.0:** ✅ **ЗАКРЫТ**

**Пересмотр:** `async_flush_interval_sec=5` (config_tuner) обеспечивает flush каждые 5 секунд. Рабочие данные (brief, теги, якоря, precision, embedding) защищены с окном потери ≤5 секунд.

| Данные | Когда в SQLite | Потеря при краше |
|--------|---------------|-----------------|
| Теги, brief, Score, embedding | каждые 5s | ≤5 секунд |
| Якоря, precision_log | каждые 5s | ≤5 секунд |
| content_full (полный текст) | конец сессии | Вся сессия — но это подстраховка, не рабочие данные |
| step_log (process_vec) | каждые 20 шагов | ≤20 сообщений |

`content_full` — страховочная копия сырого текста для ctx.full(). Не рабочий элемент системы.

### WP-03 — Конкурентная запись SQLite
**v2.0:** 🔴 → ⚠️ Частично
**v3.0:** ✅ **ЗАКРЫТ**

**Решение:** Одна SQLite WAL. Точка.

**Обоснование:** Mnemostroma = single-agent sidecar на одной машине. Observer пишет батчами каждые 5s. Agent Content пишет при content.save() (редко). Человек физически не может генерировать нагрузку при которой SQLite WAL станет бутылочным горлом. Даже при 1 msg/секунду — это 1 batch/5s. SQLite обрабатывает тысячи INSERT/s.

Вариант с двумя БД или единой очередью — overengineering для single-agent архитектуры.
