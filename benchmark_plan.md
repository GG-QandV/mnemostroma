# Mnemostroma — Benchmark Plan
> Version: 1.0 | 2026-04-01

---

## 1. Цель
Объективное сравнение эффективности Mnemostroma с существующими решениями управления памятью (MemGPT/Letta, Zep, Mem0) по метрикам точности поиска, потребления ресурсов и задержки.

## 2. Методология
- **Dataset**: 100 синтетических сессий, сгенерированных LLM, покрывающих 5 доменов (код, творчество, бизнес-анализ, личное планирование, техподдержка).
- **Ground Truth**: для каждого запроса в наборе тестов определен "идеальный" набор связанных сессий (1-3 сессии).
- **Hardware**: CPU-only environment (8 vCPU, 16GB RAM), имитация типичного ноутбука разработчика.

## 3. Метрики (KPI)

| Метрика | Описание | Цель (Mnemostroma) |
|---------|----------|---------------------|
| **Precision@5** | Доля релевантных результатов в топ-5 выдаче | > 0.85 |
| **Latency p50** | Время ответа `ctx.semantic()` | < 25ms |
| **Latency p99** | Худшее время ответа | < 100ms |
| **RAM Usage** | Пиковое потребление RSS | < 700MB |
| **Storage Amplification** | Размер БД к размеру сырых логов | < 1.2x |

## 4. Сравниваемые системы
1. **Mnemostroma** (v1.6.2+)
2. **MemGPT (Letta)** (v0.x)
3. **Zep** (Long-term memory for AI)
4. **Mem0** (Personalized memory)
5. **Baseline**: Naive sliding window (last 20k tokens)

## 5. Сценарии тестирования
- **T1: Поиск по факту** ("Какой порт мы выбрали для Redis в марте?")
- **T2: Поиск по концепции** ("Напомни философию нашего API")
- **T3: Поиск при наличии конфликтов** ("Мы передумали использовать Docker?")
- **T4: Нагрузочный тест** (1000 сессий в БД, проверка деградации Latency)

## 6. Отчетность
Результаты публикуются в `docs/benchmark_results.md` и выносятся в README.
