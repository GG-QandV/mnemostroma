## 1. Что такое режим разработчика

Chrome Web Store позволяет устанавливать расширения только после проверки Google.
**Режим разработчика** — официальная альтернатива: расширение загружается напрямую
из папки на вашем компьютере, без публикации в маркете.

Это **не обход защиты** — Chrome поддерживает этот режим официально для разработчиков
и продвинутых пользователей. Расширение работает полностью локально: никакие данные
не покидают ваш компьютер.

## 3. Шаг 1 — Получить файлы расширения

**Вариант A — клонировать репозиторий:**
```bash
git clone https://github.com/GG-QandV/mnemostroma.git
cd mnemostroma/browser_extension
```

**Вариант B — скачать ZIP:**
1. Открыть [https://github.com/GG-QandV/mnemostroma](https://github.com/GG-QandV/mnemostroma)
2. Нажать **Code → Download ZIP**
3. Распаковать архив
4. Перейти в папку `browser_extension/`

Папка расширения должна содержать файл `manifest.json` в корне.

---

## 4. Шаг 2 — Запустить демон Mnemostroma

Расширение отправляет данные на локальный демон. Без него иконка будет красной.

```bash
# Проверить что демон запущен
curl http://127.0.0.1:8766/health
# Ожидаемый ответ: {"status":"ok","mcpConfirmed":true}
```

Если демон не запущен — см. [основное руководство по установке](https://github.com/GG-QandV/mnemostroma#readme).

## 5. Шаг 3 — Загрузить расширение в браузер

### Chrome / Chromium / Edge

1. Открыть в адресной строке:
   - Chrome: `chrome://extensions/`
   - Edge: `edge://extensions/`
   - Chromium: `chromium://extensions/`

2. Включить **"Режим разработчика"** (переключатель в правом верхнем углу)

   ![Developer mode toggle](https://i.imgur.com/placeholder.png)
   *(переключатель появится после перехода на страницу расширений)*

3. Нажать **"Загрузить распакованное"** / **"Load unpacked"**

4. В диалоге выбора папки — указать папку расширения (ту, где лежит `manifest.json`):
   ```
   /path/to/mnemostroma/browser_extension/
   ```

5. Расширение появится в списке с иконкой Mnemostroma и надписью **"Mnemostroma 1.0.0"**

6. Закрепить иконку в тулбаре: нажать иконку пазла (расширения) → закрепить Mnemostroma

---

### Firefox

1. Открыть в адресной строке: `about:debugging`

2. Нажать **"This Firefox"** / **"Этот Firefox"**

3. Нажать **"Load Temporary Add-on…"** / **"Загрузить временное дополнение…"**

4. Выбрать файл **`manifest.json`** внутри папки расширения (не папку, а именно файл)

5. Расширение загрузится и появится в списке

> **Важно для Firefox:** временное расширение удаляется при перезапуске браузера.
> Для постоянной установки без маркета — использовать
> [Firefox Developer Edition](https://www.mozilla.org/firefox/developer/) или
> [Firefox ESR с подписанным XPI](https://extensionworkshop.com/documentation/publish/signing-and-distribution-overview/).

## 6. Шаг 4 — Проверить установку

1. **Иконка в тулбаре** — нажать на иконку Mnemostroma

2. **Статус демона:**
   - 🟢 зелёный — демон запущен, всё работает
   - 🟡 жёлтый — демон работает, но захват отключён
   - 🔴 красный — демон недоступен (проверить `127.0.0.1:8766/health`)

3. **Проверить захват:**
   - Перейти на [claude.ai](https://claude.ai) или [chatgpt.com](https://chatgpt.com)
   - Отправить любое сообщение
   - Иконка должна позеленеть после ответа

4. **Убедиться что данные попадают в демон:**
   ```bash
   curl http://127.0.0.1:8766/health
   ```

## 7. Обновление расширения

При обновлении файлов расширения (например, после `git pull`):

**Chrome / Edge:**
1. Открыть `chrome://extensions/`
2. Найти Mnemostroma → нажать кнопку 🔄 (обновить)

Или нажать **"Обновить"** в верхней части страницы `chrome://extensions/` — обновит все расширения.

**Автообновление через командную строку:**
```bash
git -C /path/to/mnemostroma pull
# затем нажать 🔄 в chrome://extensions/
```

**Firefox:** перезагрузить временное расширение в `about:debugging` → кнопка **"Reload"**.

## 8. Устранение проблем

### Иконка красная сразу после установки

Демон не запущен или порты заняты.
```bash
curl http://127.0.0.1:8766/health   # должен вернуть {"status":"ok","mcpConfirmed":true}
```

### Chrome показывает предупреждение "Расширение не из магазина"

Это нормально для расширений в режиме разработчика. Нажать **"Отключить"** не нужно —
предупреждение появляется при каждом запуске Chrome и не влияет на работу расширения.

Чтобы убрать уведомление: Chrome не позволяет его отключить для сторонних расширений —
это намеренное ограничение безопасности.

### Расширение исчезло после перезапуска (Firefox)

Временные расширения Firefox удаляются при закрытии браузера. Загрузить повторно через
`about:debugging` → **"Load Temporary Add-on…"**

### Данные не захватываются на Gemini

Gemini требует `world: MAIN` для перехвата fetch-запросов (нужно для определения ID беседы).
Убедитесь что в `chrome://extensions/` расширение показывает статус **"Включено"** (не "Ошибка").

### Ошибка "Could not load manifest"

Папка выбрана неправильно. Убедитесь что в выбранной папке есть файл `manifest.json`:
```bash
ls /path/to/browser_extension/manifest.json
```
